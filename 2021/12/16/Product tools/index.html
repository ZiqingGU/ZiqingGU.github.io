Posted on 2021-12-16 In Code tips

我的coding工具们：

  1. 生产工具

    • 图形化界面

      Pycharm：专业维护，功能强大，建议用专业版，但是需要license，学生邮箱可以注册使用一年，问题是经常容易out of memory，适合新手

      Jupyter notebook: 适合打草稿，每输入一行可以运行一行，适合入门，缺点是要通过kernel选择环境，经常会运行不出来环境

    • 命令行界面

      普通terminal：适合进阶，通常用vim + xx.py写入和修改代码，pdb添加断点调试，缺点是窗口关闭运行程序也会结束

      工具terminator：多个窗口

      tips:可以用screen或者tumx设置多个窗口，使用时进入窗口，不会因为突然断开而导致程序中断

  2. 生产环境

    本地常用anaconda，在命令行或者图形化界面都可使用

    无图形化界面的多人服务器适合采用docker，就好比手机上的应用软件，下载下来不需要运行编译，直接可以安装使用。通过docker image（镜像） 建立不同的容器，容器之间不会相互影响。本地也可以用docker，本地支持可视化编译器，因此可以将代码文件挂载到相应的docker中，可以实现Pycharm写代码而不需要vim进入，容器端无缝运行。

    在docker中运行代码要格外注意路径，区分os.path和sys.path，路径有问题时，Python解释器往往会报错

    • 创建容器

  3. 代码托管

    github或者gitlab

    与本机连接需要isa-key

    • git 常用命令

  4. 文件传输

    ifconfig查看本机ip

    docker 与宿主机之间传输用docker cp

    节点与节点间传输用scp
